# Forms, Navigation & Search

## Form Design
- Reduce the number of fields to the absolute minimum.
- Always use a visible label above every field — never rely on placeholder text as the only label.
- Left-align all labels and fields for readability.
- Validate on blur (when leaving a field), not on every keystroke. (Exception: Password strength in real-time).
- Match input width to the expected content length.
- Use the correct input type: email, tel, number, date, password.
- Inline error: red border, error icon, error message below the field.

## Navigation Patterns
- **Top navigation bar**: Best for 3-7 primary sections.
- **Side navigation**: Best for complex products (dashboards). Highlight active item. Max 2 levels nesting.
- **Bottom navigation**: Best for mobile (3-5 items).
- Always indicate the current location (active state).

## Search UX
- Show suggestions after 2 characters of input.
- Limit to 5-8 suggestions visible without scrolling.
- Never show a blank page for zero results. Suggest alternative queries or clear filters.
- Allow sorting and filtering on search results pages.

## Onboarding & First-Run
- Minimize steps to first value.
- Empty states are onboarding opportunities — design them intentionally with an illustration and one primary CTA.
- Use contextual tooltips the first time a user encounters a feature, rather than blocking the app with a forced tour.
